# cn-core-paper

一个面向中文核心工程期刊论文写作、改稿与投稿前精修的 Codex skill。

这个仓库的目标，不是单纯提供几份写作模板，而是把“中文核心工程论文”的常用改稿逻辑，整理成一个可以被 Codex 直接调用的 skill。  
它适合处理系统设计、仪器测试、嵌入式实现、数据采集、测控、电子信息、自动化等偏工程类稿件。

## 这项 skill 能做什么

- 把初稿改成更像中文核心工程期刊的风格
- 按正式论文章节重构项目汇报式稿件
- 统一题目、摘要、关键词、结论、术语和单位
- 收紧语言，减少明显 AI 味和项目汇报味
- 规范图表、公式、Word 排版口径
- 做投稿前总检查

## 适用场景

- 中文核心论文初稿写作
- 工程类论文精修
- 论文主线重构
- 投稿前终检
- 让 AI 协助改稿时的统一约束模板

## 最简单的用法

在 Codex 中直接调用：

```text
Use $cn-core-paper 把这篇稿子改成中文核心工程期刊风格，保持数据口径不变。
```

或者：

```text
Use $cn-core-paper 按中文核心工程论文结构重写这篇文章，不要写成项目汇报。
```

或者：

```text
Use $cn-core-paper 对这篇文章做投稿前终检，重点查摘要、结论、图表、公式、单位和术语一致性。
```

## 核心特点

- 先保主线，再改语言
- 先保数据口径，再动结构
- 先像正式期刊论文，再做润色
- 强调工程证据、结果参数和表达边界
- 不把论文写成项目汇报或 AI 综述

## 适合谁用

- 研究生
- 科研人员
- 高校教师
- 工程师
- 技术负责人
- 需要批量整理论文表达模板的人

## 安装方式

把本仓库文件夹放到本地 Codex skills 目录，例如：

```text
C:\Users\<YourName>\.codex\skills\cn-core-paper
```

然后在 Codex 中直接使用 `$cn-core-paper` 即可。

## 说明

这项 skill 偏“工程类中文核心期刊”写作，不默认服务于文学、管理学、法学等非工程方向稿件。  
如果你的文章同时面向中文核心和 SCI/英文稿，建议再做一层期刊目标区分，不要直接混用同一套语言风格。
