# T1级工科绘图审美与复现规范

本文件用于指导中文核心工程论文中的高质量图件设计、复现和审美升级。目标不是只满足“能看懂”，而是让图件同时覆盖 T2/T3 规范要求，并具备冲击 T1 类中文核心期刊的图形成熟度。

## 1. 总目标

默认把图件分为三个层级处理：

- **T3 合格图**：术语、单位、图题、坐标轴、编号正确；黑白打印可读；没有明显重叠。
- **T2 稳妥图**：结构清晰、线宽统一、版式规整；图中信息能直接支撑正文结论；适合大多数中文核心工程期刊。
- **T1 竞争图**：在 T2 基础上进一步具备顶级工科期刊审美：结构严格、视觉层级克制、可复现数据路径完整、图组叙事清楚、缩印后仍有辨识度。

处理图件时默认向 T1 靠拢，除非用户明确只要快速投稿版或草图版。

## 2. T1 图件的硬标准

T1 级图件必须同时满足：

1. **结构正确**：系统图、拓扑图、等效电路、机构爆炸图、参数分区图不能为了好看牺牲物理或数学关系。
2. **图形克制**：少用装饰性阴影、渐变、大色块、图标化插画和 PPT 风格高亮。
3. **层级明确**：主结构最突出，辅助线弱化，引线和网格更淡，文字只保留必要信息。
4. **黑白可读**：即使去掉颜色，仍能通过线型、灰度、标记形状、区域边界区分类别。
5. **缩印可读**：按期刊单栏或双栏宽度缩小时，坐标轴、器件编号、关键标注仍可辨识。
6. **数据可追溯**：曲线、曲面、误差棒、分区图要能追溯到数据、公式、仿真或明确的示意数据，不能只画视觉效果。
7. **图组有叙事**：多子图应形成“结构/方法 -> 机理/模型 -> 结果/对比”的顺序，而不是把零散图片拼在一起。

## 3. 默认视觉参数

若目标期刊无明确模板，默认采用以下范围：

- 字体：中文宋体或等效宋体；英文、数字、变量用 Times New Roman；变量和物理量按需要斜体。
- 图内字号：最终版缩印后不低于可读阈值；绘图源文件中常用 7.5-10 pt 作为论文图基准。
- 主轮廓线：0.8-1.2 pt。
- 辅助线、引线、网格线：0.35-0.7 pt。
- 坐标轴线：0.8-1.0 pt。
- 曲线线宽：0.8-1.2 pt；多曲线比较时优先用不同 marker 和线型，而非只靠颜色。
- 颜色：默认黑、灰、浅灰；需要分组时使用低饱和蓝、绿、橙、红、紫，且不超过 5-7 个主要类别。
- 背景：白底优先；避免深色背景，除非目标期刊或图件类型明确需要。

## 4. 工具链策略

不要用一个工具硬画所有图。按图件类型选工具：

- **系统图、流程图、拓扑图、等效电路、建模框图**：优先 Visio / PPT / Illustrator / SVG 脚本。重点是对齐、吸附、统一线宽、可编辑文本。
- **标定曲线、误差棒图、i-c 曲线、性能比较图**：优先 Python / MATLAB / Origin。重点是数据口径、坐标轴、图例、误差棒、导出一致性。
- **谱图、三维曲面、仿真云图、参数扫描图**：优先 MATLAB / Python。重点是坐标系统、色标、刻度、视角和可复现数据。
- **机构爆炸图、装配图、三维结构图**：优先 CAD / SolidWorks / Fusion / Illustrator / PPT 后处理。没有 CAD 模型时可用矢量重绘，但必须保留装配轴、部件关系和标注逻辑。
- **复杂拼版**：最终回到 PPT / Illustrator / LaTeX / SVG 统一版式。

交付时优先采用 vector-first 工作流：源文件 -> SVG/PDF -> 高分辨率 PNG。不要先从低清位图上修。

## 5. 复现图的边界

复现论文图时，采用“重绘式复现”，不要逐像素复制原图。

必须保留：

- 原图的问题对象、子图结构、图号关系；
- 关键模块、变量、器件、坐标轴、图例；
- 原图的图形语言，例如拓扑图的器件对称、MPC 图的硬边界分区、谱相干图的三维坐标和色标。

可以改进：

- 线宽、灰度、字体、标注拥挤、图例位置；
- 分辨率、矢量化程度、版式均衡；
- 低质量截图中的噪声和不清晰符号。

不能改成：

- 自己发明的新图；
- 与原论文图表达对象不同的泛化示意；
- 只保留主题词但结构关系被替换的“AI 插画”。

## 6. 八类代表图的 T1 处理规则

### 6.1 概念装备图：BLEEX 类

核心是“人体/装备/载荷/关节/引线标注”的概念组织。

- 人体轮廓要淡，外骨骼杆件和关节要突出。
- 髋、膝、踝节点必须形成清楚的运动链。
- Payload、Power Supply & Computer、Harness 等标签应水平或短折线引出，尽量不交叉。
- 允许浅灰填充载荷和电源模块，但不要做宣传图式渲染。

T1 判断：一眼能看出机械系统如何绕人体布置，而不是只看到一个人物插画。

### 6.2 电力电子系统图与拓扑图：PV-DCMLI、风电全功率变流类

核心是器件规范、母线对齐、相桥臂对称和变量标注。

- 直流母线必须水平或垂直严格对齐。
- 每相桥臂开关中心线、二极管方向、编号顺序必须统一。
- Va/Vb/Vc、ia/ib/ic 等输出点要从同一参考高度或清晰节点引出。
- 系统级方框图与详细拓扑图应在视觉上有主次关系，不能互相抢。
- 风电背靠背变流器要明确 generator rectifier、DC link、grid inverter、filter、utility grid。

T1 判断：即使删掉图题，审稿人也能通过结构判断这是规范的电力电子拓扑，而不是普通框图。

### 6.3 传感器原理图 + 等效电路 + 标定曲线

核心是“结构 -> 工作机理 -> 等效电路 -> 数据验证”的闭环。

- 结构图要清楚标出 sensing region、pad、interconnect、关键几何尺寸。
- 工作状态对比图要使用相同视角和比例，例如 untouched / touched。
- 等效电路符号必须规范，交流源、电容、接地、人体等效支路位置要清楚。
- 标定曲线要有单位、图例、误差棒或数据点说明。
- 仿真值和实验值要用不同 marker/线型区分，不要只靠红蓝颜色。

T1 判断：图组本身能说明传感器为什么有效，以及数据如何支持机理。

### 6.4 机构爆炸图与单支路运动学图：LVSA 类

核心是装配轴、部件分组、空间层次和力学抽象。

- 爆炸图应沿同一装配轴线展开，零件间距表达装配关系。
- Unit A / Unit B 分组可以用浅边框，但主部件不能被边框抢走。
- 弹簧、滑块、连杆、导轨、位置限位必须有清楚的接触或连接关系。
- 单支路示意图要把旋转关节、连杆、滑块、弹簧轴线画成同一套运动学语言。
- 颜色只用于部件分组，饱和度要低。

T1 判断：不看正文也能区分“真实部件展示”和“等效支路模型”两层含义。

### 6.5 显式 MPC / 参数空间分区图

核心是多面体区域、硬边界、参数轴和截面关系。

- 区域必须是硬边界多边形/多面体，不能画成软过渡热图。
- 极窄区域也要保留，不要为了美观合并。
- 3D 图中透明度要统一，遮挡关系要清楚。
- 2D 截面图要保留网格和轴标签，区域颜色低饱和。
- 最好保存 region vertices 或生成代码，保证可复现。

T1 判断：图看起来像控制理论中的显式解分区，而不是装饰性色块。

### 6.6 谱相干图、时频诊断图和三维仿真图

核心是坐标系统、色标、峰脊结构和测量解释。

- cyclic frequency、carrier frequency、coherence 等轴标签要完整。
- 色标必须有范围，例如 0-1。
- 峰脊或能量带应符合诊断对象的周期结构，不能只做随机纹理。
- 红色箭头或注释只能用于关键间隔，例如 Delta alpha、Delta f。
- 三维视角要服务信息读取，不追求夸张透视。

T1 判断：图像首先是测量/诊断结果，其次才是好看的三维图。

### 6.7 微电极阵列工艺流程图 + i-c 曲线

核心是工艺过程、局部放大、几何尺寸和电化学响应的对应关系。

- 工艺图中 deposition head、UV ray、substrate、trace 等要清楚。
- 局部放大框、连接线、尺度尺要像显微图语言。
- 几何尺寸如 trace spacing、electrode width 必须直接标注。
- i-c 曲线要突出斜率差异；噪声曲线要突出基线波动和噪声水平。
- 同一图组中工艺图和曲线图的字体、线宽、子图编号要统一。

T1 判断：工艺参数与响应结果之间有可见联系。

### 6.8 系统建模框图

核心是模块边界、变量流向和模型层级。

- 方框大小按层级统一，模块名称居中。
- 主流程箭头与反馈箭头应有明确方向。
- 变量标注贴近箭头，不悬空。
- 若上半是物理系统、下半是模型框图，应共享术语和变量。

T1 判断：读者能看懂模型如何从物理系统抽象出来。

### 6.9 卷积神经网络结构图：模块化论文插图样式

最近效果最稳定、也最贴近中文核心审美的一类 CNN 图，不是 3D 立体块风格，而是“论文式模块流水线图”。
它的核心是结构清楚、分组克制、图例统一、白底可印。

- 版式优先于颜色：从左到右排列输入、卷积块、池化块、全连接层、输出层，主流程一条线走到底。
- 每个 stage 使用虚线圆角分组框，顶部写模块名，底部写重复次数或阶段说明，例如 `x2`、`x3`。
- 模块内部只保留必要注释，例如 `Conv 3x3`、`BN`、`ReLU`、`MaxPool 2x2`，不要堆叠长句。
- 特征提取、特征表示、分类预测等阶段用底部横向箭头串起来，箭头比文字更重要。
- 图例放到底部统一解释颜色和模块类型，避免在主体图中重复解释。
- 颜色只做分组标识，推荐低饱和蓝、绿、红、紫、棕，且避免大面积高饱和。
- 输出层、FC 层、Dropout、Softmax 等要和主干分开，不要让右侧分类头抢走主体视觉。
- 中文标题和英文图题都应短、准、稳，像正式期刊图题，而不是展示页标题。
- 矢量优先，最终应能导出 SVG/PDF；PNG 只做预览或投稿替代格式。

可执行判断：

1. 读者能否一眼看出 CNN 的层级和重复结构。
2. 去掉颜色后是否还清楚知道卷积、池化、全连接和输出的分工。
3. 黑白打印时是否仍能通过虚线框、线型和图例区分模块。
4. 图题是否像论文图题，而不是教程海报标题。
5. 该图能否直接作为中文核心论文中的结构示意图复用。

### 6.10 多面板综合示意图：材料、催化、环境、生物类总图

这类图最接近顶级中文核心期刊常见的“总图”气质。它不是单幅插画，而是把制备、结构、机理、性能、验证串成一个完整叙事。

- 常见版式是 3-4 个主面板横向排布，顶部统一用深蓝或低饱和色圆角标题条编号。
- 每个主面板内部再分若干子区块，外层用细边框或虚线框限定，内部用浅色底和短箭头串联。
- 左侧常放制备/采样/流程，中间放微观结构或机理，右侧放性能曲线、柱状图、验证图或治理框架。
- 图中可以有少量轻微体积感，但必须像论文里的矢量示意图，不像 AI 渲染图、商品渲染图或海报插画。
- 白底、低饱和、浅蓝/浅绿/浅紫/浅粉是主色系，颜色只承担分区和引导作用。
- 允许图标化的烧杯、反应釜、细胞、组织、地图、水体、孔道、球形颗粒、层状结构、箭头和统计图，但要统一线宽和光照逻辑。
- 文字宜短，不堆说明句；长解释放到图注或正文，图内只留最必要标签。
- 右侧曲线图、柱状图、饼图要像正式论文插图，不要做成演示文稿截图。
- 如果图里有 3D 结构，只保留“示意性的体积感”，不要保留真实质感、镜面反射、颗粒噪点、背景虚化和夸张阴影。

T1 判断：读者一眼能看出这是“制备/结构/机理/性能”的总图，而且黑白打印后仍然清楚。

## 7. 与 deep research 和 image2 的配合

当用户希望结合深度研究和 image2 画图时，按以下顺序：

1. **深度研究先定图源**：列出原论文、图号、图题、图类型、关键对象、版权边界。
2. **Codex 先做结构拆解**：把图拆成模块、坐标轴、器件、子图、标注和数据。
3. **先生成矢量骨架**：系统图、拓扑图、流程图、等效电路优先 SVG/PPT/Visio 风格。
4. **数据图用代码生成**：曲线、误差棒、分区、谱图必须保留生成脚本或数据表。
5. **image2 只用于位图型素材**：例如概念草图、真实感材质、显微图风格占位、封面型展示。不要让 image2 直接决定拓扑、电路或坐标轴。
6. **最终统一版式**：将图件回收到同一画布，统一字体、线宽、子图编号和图题口径。

给 image2 的提示词必须明确：“保持论文插图气质、低饱和、白底、矢量感、不要海报风、不要炫光、不要复杂背景”。涉及电路、拓扑、坐标轴和公式的关键部分，不依赖 image2 自动生成，必须由 Codex 或绘图软件控制。

## 8. 投稿前图件审美检查清单

投稿前逐项检查：

- 图号、图题、正文引用完全一致。
- 所有坐标轴都有物理量和单位。
- 图例名称与正文术语一致。
- 曲线颜色、marker、线型在黑白打印下仍可区分。
- 子图编号位置统一，例如 (a)、(b)、(c)。
- 引线不交叉或尽量少交叉。
- 文本不压线、不压点、不压器件。
- 线宽没有忽粗忽细。
- 网格线足够淡，不抢曲线。
- 电路/拓扑/机构符号符合工程习惯。
- 图中没有长句解释，解释放正文。
- 位图分辨率足够，矢量图没有锯齿和低清截图痕迹。
- 若为复现图，已确认不是直接复制原图，且保留原论文引用。

## 9. 默认响应策略

当用户要求“画图”“复现图”“提升中文核心图件”“T1/T2/T3 图件审美”时：

1. 先判断图件类型和目标层级。
2. 若有原论文或深度研究报告，先提取原图号、子图结构和关键标注。
3. 先给结构拆解，再开始画，不要直接生成泛化新图。
4. 对曲线、分区、谱图保留脚本；对系统图和机构图保留可编辑矢量源。
5. 输出时说明当前版本属于 T3/T2/T1 哪个层级，以及下一步怎样升到更高层级。
